# 辱诺大杂烩 — 技术分析文档

## 运行环境

### 宿主框架

本脚本运行于 **LSPosed** 模块框架之上。LSPosed 是一个基于 Riru/Zygisk 的 Android Xposed 框架实现，可在不修改 APK 的前提下向目标应用注入代码。

### 模块载体

脚本的实际宿主模块名为「**模了个块**」，该模块通过 LSPosed 挂载到 QQ（`com.tencent.mobileqq`）进程中，在 QQ 运行时提供以下能力：

- 内置 BeanShell 脚本引擎，可动态加载和执行 Java 语法子集的脚本文件
- 拦截 QQ 内部消息事件，将消息对象分发给脚本的 `onMsg` 回调
- 封装 QQ 内部 API 为脚本可调用的全局函数（禁言、撤回、改名片、发送消息、表情回应等）
- 提供脚本生命周期管理（加载、停止、卸载回调 `onUnload`）
- 为每个脚本分配独立的存储空间（`setData` / `getData` / `getInt`）

### 脚本加载目录

/storage/emulated/0/Android/media/com.tencent.mobileqq/.模了个块/Plugin/

每个脚本以子目录形式存放，目录内至少包含：
- `main.java` — 主脚本文件（BeanShell 语法）
- `info.prop` — 脚本元信息（名称、作者、版本）
- `desc.txt` — 脚本描述（Markdown 格式，在模块界面展示）

模块界面中点击脚本即可加载执行，停止脚本会触发 `onUnload` 回调。

### 开发文档

本脚本严格依据模块提供的开发文档编写。所有调用的 API（`mute`、`recall`、`send`、`setCard`、`members`、`emoji`、`sleep`、`log` 等）均为文档中列出的官方接口。文档中还列出了 BeanShell 的语法限制，本脚本全程避开了以下不支持的特性：

- 注解（Annotation）
- 枚举（enum）
- Lambda 表达式 / Stream API
- 泛型类型参数（写 `List` 而非 `List<String>`）
- 访问修饰符（`public` / `private` 会被自动移除）

### 核心执行线程说明

`onMsg` 回调在 QQ 主线程（UI 线程）上执行，因此本脚本遵循以下原则：

- **表情轰炸**：`emoji()` 为文档内置函数，瞬间发射 23 个表情，耗时极短，主线程直接执行
- **禁言**：`mute()` 为文档内置函数，同步调用，主线程直接执行
- **延迟连招（辱骂+发图+撤回）**：全部放入独立 `Thread` 中执行，避免 `sleep` 阻塞 QQ 主线程导致 ANR/闪退
- **名片守护**：独立后台 `Thread`，每 5 秒主动扫描群成员列表并修正目标名片

## 执行流程

目标用户发送消息 → `onMsg` 事件触发（主线程）→ 依次执行：

### 第一阶段：主线程内即时完成
1. 表情轰炸 1~23 号表情（`emoji()` × 23，瞬间发射）
2. 随机时长禁言（`mute()`，同步调用）
3. 提取目标消息 msgId（反射读取）

### 第二阶段：独立线程异步执行
4. 等待 3~6 秒随机延迟
5. 发送前置辱骂（随机抽取，文字/图片各有独立词库）
6. 等待 3~6 秒随机延迟
7. 发送后置辱骂（随机抽取）
8. 发送预设图片 `NuoHeYin.jpg`
9. 等待 10~30 秒随机延迟
10. 精准撤回目标消息（`recall()`）

### 附带任务：名片守护
- 独立后台线程，每 5 秒扫描一次群成员列表
- 比对目标用户当前群名片与预设值
- 不一致则立即通过 `setCard()` 覆盖

## 消息类型识别

由于 BeanShell 运行时无法直接访问 `msg.isImg` 属性（该字段在某些 QQ 版本中仅通过 getter 暴露），本系统采用反射多层兜底方案：

- 第一层：反射获取 `isImg` 字段
- 第二层：反射调用 `isImg()` 方法
- 第三层：反射调用 `getIsImg()` 方法
- 第四层：通过 `msgType` 枚举值推断（3=图文, 7=视频视为图片类；1=文字时通过 Content 字段前缀辅助判断是否为图片路径）

该方案确保在不同 QQ 版本和消息对象封装层级下均能正确识别消息类型。

## 词库系统

系统内置四个独立词库，每个词库 50 条内容，分为两级发送：

| 词库 | 触发条件 | 条数 | 攻击方向 |
|------|---------|------|---------|
| preTextDict | 文字消息前置 | 50 | 攻击智力缺陷、语言组织能力、先天残疾 |
| postTextDict | 文字消息后置 | 50 | 攻击直系亲属生殖器官形态、分泌物、味道 |
| preImgDict | 图片消息前置 | 50 | 攻击截图能力、视觉障碍、手眼协调 |
| postImgDict | 图片消息后置 | 50 | 攻击直系亲属生殖器官形态、分泌物、味道 |

前置词库与目标用户的行为直接相关（语言能力/截图能力），后置词库为无厘头猎奇辱骂。每条消息触发时从前置和后置词库各随机抽取一条，组合成两连发。

## 名片守护

通过定时检查群成员列表，比对目标用户当前名片与预设值是否一致。若不一致则立即通过 `setCard()` 覆盖。检查间隔可配置，默认 5 秒。使用 `members(groupId, false)` 从缓存读取，性能开销极小。

成员字段读取使用反射多层兜底方案，路径为：
1. 优先通过 nickInfo 对象读取（新版 API）：`nickInfo.troopnick` → `nickInfo.nick`
2. 回退到旧版反射方法：`getTroopnick()` / `troopnick` 字段
3. QQ 昵称回退链：`nickInfo.nick` → `autoremark` → `friendnick`
4. 最后尝试 `Map.get()` 兜底

控制台展示昵称优先级：群名片（troopnick） → QQ 昵称 → QQ 号。名片守护的后台比对逻辑仍仅比对群名片字段，不受展示逻辑影响。

---

## 消息撤回

撤回功能使用框架内置 `recall(contact, msgId)` 接口。消息 ID 通过反射从 `msg.msg` 原始消息对象中提取，遍历多个可能的字段名（`msgId`、`shmsgseq`、`uniseq`、`msgSeq`、`msgUid`）和方法名（`getId`、`getMsgId`、`getShmsgseq`），确保兼容不同 QQ 版本的消息对象封装。

撤回延迟为随机 10~30 秒，在独立线程中执行，不影响 QQ 主线程。

---

## 触发计数

使用 `getInt` / `setData` 持久化存储触发次数，脚本重启后计数不丢失。每次完整执行后计数加一并写入存储。日志仅记录触发序号、禁言时长、消息类型和撤回状态，保持简洁。
