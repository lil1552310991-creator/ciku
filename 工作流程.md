# 辱诺大杂烩 — 开发工作流程

> 每次修改 `main.java` 时，必须同步更新以下文件，保持文档与代码一致。

---

## 必须同步的文件

| 序号 | 文件 | 何时更新 | 更新内容 |
|------|------|---------|---------|
| 1 | `main.java` | 每次改代码 | 实现功能 / 修 bug |
| 2 | `info.prop` | 功能变更时 | `version=` 加一号 |
| 3 | `desc.txt` | 功能变更时 | 版本号、功能介绍、更新说明、使用步骤 |
| 4 | `updlog.md` | 每次改代码 | 顶部新增版本条目，列出所有改动 |
| 5 | `用前必看.md` | 架构/API变更时 | 技术实现细节、执行流程、配置项表格 |

---

## 版本号规范

- 格式：`v主.次`，如 `v4.0`
- 大改（架构重构、新系统）→ 主版本 +1
- 小改（修 bug、小优化）→ 次版本 +1
- 当前：**v4.0**

---

## desc.txt 格式

```
# 辱诺大杂烩 vX.X

> 简述 ｜ 作者：原神哥
> 特色说明

## 功能介绍
（每项功能独立段落）

## vX.X 更新内容
- 改动列表

## v旧版 更新内容
（保留最近两个版本）

## 使用说明
（步骤）
```

## updlog.md 格式

```
# vX.X (YYYY-MM-DD) — 简短标题

- 改动1
- 改动2

---

# v旧版 (日期)
（保留全部历史）
```

## 用前必看.md 更新要点

- 执行流程变化 → 更新「执行流程」章节
- 配置项变化 → 更新「配置项说明」表格
- 技术实现变化 → 更新对应技术章节
- API 调用方式变化 → 更新对应说明
